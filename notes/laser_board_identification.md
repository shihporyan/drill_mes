# 雷鑽板子辨識 — 利用宣告夾帶板名

**主旨：** 我們想知道雷射鑽孔機在鑽什麼板子。

## 背景

每個 `.lsr` 檔案都會有一行「宣告」，雷鑽機透過這個格式知道要鑽什麼。

範例：

```
OSC46BC0.0/0/0/0.0/0/[G001/0/0/0]/UP/Count:114:0.35:0
```

宣告各欄位拆解：

| 區段 | 意義 | 可否修改 |
|---|---|---|
| `O` | 機台指定起始字元 | ❌ 不可改 |
| `SC` | 板材代號 | ✅ 操作手可改 |
| `46` | 孔徑 | ✅ |
| `B` | 板子名稱（A～F 板） | ✅ |
| `C0.0/...` | 機台指定參數 | ❌ 不可改 |

## 關鍵發現

從 `O` 到 `C0.0/...` 之間的字串長度沒有限制，可任意加字元而不違反宣告規則。

例：`OSC46BC0.0` → `OSC46ABCDEFGHC0.0`（中間插入 `ABCDEFG`）

→ 表示我們可以在這個區段「夾帶板子名稱」資訊，雷鑽機照常解析。

## 現況問題

操作手不會每片板子都在宣告中打板子名稱，例如 D/E/F 板就常常沒改。
→ 導致 LOG 解析時無法可靠判斷實際在鑽哪一塊板子。

## 後續行動

與雷鑽操作手討論：是否能要求每次都在「宣告」中打上板子名稱（即使是 D/E/F 板），讓 LOG 端可以一致地辨識板子。
